## 📅 [2주차] 사전 개념 학습 및 API 설계 (26.01.23 ~ 26.01.29)

### 📚 사전 개념 학습 (Deep Dive)

#### ■ 1. HTTP (HyperText Transfer Protocol)
서버와 클라이언트가 데이터를 주고받기 위해 사용하는 통신 규약입니다.

**1.1 통신 구조 및 프로세스**
> 브라우저 주소창에 `https://example.com/rest/login/202601` 입력 시 시나리오:
1. **DNS 조회**: 도메인 이름을 실제 서버 IP 주소로 변환 요청.
2. **TCP/IP 연결**: 3-Way Handshake를 통해 브라우저와 서버 간 신뢰성 있는 통신 통로 생성.
3. **HTTP 요청 전달**: 생성된 통로를 통해 브라우저가 HTTP 요청 메시지를 서버에 전송.
4. **서버 로직 처리**: 서버에서 DB 데이터 추출 등 비즈니스 로직 수행.
5. **HTTP 응답 생성**: 처리 결과를 JSON 등 형식으로 변환하여 응답 메시지 전송.
6. **브라우저 렌더링**: 받은 데이터를 사용자 화면에 출력.

**1.2 주요 특성**
- **비연결성 (Connectionless)**: 응답 후 즉시 연결을 해제하여 서버 자원을 효율적으로 사용하나, 매번 연결 설정 시 오버헤드 발생. (현대 HTTP에서는 `keep-alive`로 보완)
- **무상태성 (Stateless)**: 서버가 이전 상태를 보존하지 않아 서버 확장성(Scale-out)에 유리하나, 클라이언트가 매번 필요한 모든 정보를 담아 보내야 함. (쿠키, 세션, JWT로 보완)

#### ■ 2. RESTful API (Representational State Transfer)
HTTP의 장점을 최대한 활용하기 위한 아키텍처 스타일로, 직관적인 의도 파악을 목표로 합니다.

**2.1 REST API의 3요소**
- **자원(Resource) - URI**: 모든 것을 자원으로 관리하며, 명사를 사용해 계층 구조 표현 (예: `/rest/login/202601`).
- **행위(Verb) - HTTP Method**: 무엇을 할지 메서드로 표현.
    - `GET`(조회), `POST`(생성), `PUT`(전체 수정), `PATCH`(부분 수정), `DELETE`(삭제).
- **표현(Representation)**: 서버와 클라이언트가 데이터를 주고받는 형식 (주로 JSON).

#### ■ 3. URL 경로 표현 방식
- **PathVariable (경로 변수)**: URL 일부를 데이터로 사용. 필수 데이터 전달이나 특정 리소스 식별 시 사용 (예: `/users/100`).
- **Query String (쿼리 스트링)**: `?key=value` 쌍으로 전달. 데이터 정렬, 필터링, 검색 등 부가 옵션 설정 시 사용 (예: `/users?id=100`).
- **결론**: 리소스 식별은 **PathVariable**, 결과값 필터링 및 정렬은 **Query String**을 선택하는 것이 일반적입니다.
